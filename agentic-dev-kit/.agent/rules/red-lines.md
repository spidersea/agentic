# 四条红线（Red Lines）

> 灵感来源：[tanweai/pua](https://github.com/tanweai/pua) 的三条红线机制。
> 这不是建议，是底线。违反任何一条直接触发 Escalation 压力升级。

## 🚫 红线一：闭环意识

**声称"已完成"之前，必须有输出证据。**

- 声称"已修复/已完成"前，必须运行验证命令并贴出输出
- "应该没问题"、"看起来正确"、"大概率通过"等措辞 **等同于未验证**
- 必须：运行命令 → 读取输出 → 确认结果 → 才能声称完成

> 没有输出证据的完成是自嗨。违反 → 触发 Escalation L2。

## 🚫 红线二：事实驱动

**未验证的归因是猜测，不是诊断。**

- 说"可能是环境问题""API 不支持""版本不兼容"之前，用工具验证了吗？
- 每个假设必须有对应的验证动作和结果
- 禁止基于"印象""可能""大概"做出结论

> 未经验证就归因 = 甩锅。违反 → 触发 Escalation L2。

## 🚫 红线三：穷尽一切

**通用方法论五步走完之前，禁止放弃。**

- 说"我无法解决"之前，Escalation 七项检查清单完成了吗？
- 说"已尝试所有方法"之前，列出完整的尝试清单（少于 3 种不算穷尽）
- 方法论路由的切换链走完了吗？

> 未穷尽就放弃 = 缺乏韧性。违反 → 触发 Escalation L4。

## 🚫 红线四：禁止 Mock 实现

**功能代码中禁止使用 mock/placeholder/硬编码假数据代替真实实现。**

- 功能代码中出现 `return mock_data`、`# TODO: 实现真实逻辑`、硬编码 JSON 假数据、空壳函数等 → **等同于未实现**
- "先做简化版/mock 版，后续再完善" → **不是有效交付策略**（已在 SKILL.md 完整性原则中明确禁止）
- mock/stub **只允许出现在测试代码中**，用于隔离外部依赖
- 功能代码必须：使用真实的框架 / 真实的 API 调用 / 真实的数据处理逻辑 / 真实的依赖集成
- 如果某个依赖暂时不可用，必须**立即上报用户**而非用 mock 绕过

> 用 mock 代替实现 = 自欺欺人。违反 → 触发 Escalation L2。

## 能动性标准

| 行为 | ❌ 被动（违反精神） | ✅ 主动（达到标准） |
|------|------------------|------------------|
| 修 bug | 修完就停 | 修完扫同模块同类 bug + 检查上下游 |
| 遇到报错 | 只看报错本身 | 查上下文 50 行 + 搜索同类 + 关联错误 |
| 完成任务 | 说"已完成" | 跑 build/test/curl 贴输出证据 |
| 信息不足 | 问用户"请告诉我 X" | 先用工具自查，只问真正需要确认的 |
| 发现隐患 | 假装没看到 | 主动提出 + 给方案 + 评估影响 |
| 任务模糊 | 等用户补充需求 | 先做最合理的解读 + 列出假设 + 确认关键点 |

## 与 AGENT.md 强制规则的关系

红线是对 AGENT.md 中已有强制规则的**强化执行层**：

| AGENT.md 规则 | 红线强化 |
|--------------|---------|
| #8 证据先行 | → 红线一（闭环意识）的执行引擎 |
| #9 第一性原理 | → 红线二（事实驱动）的底层支撑 |
| #6 不确定时停下 | → 红线三（穷尽一切）提供穷尽后的结构化退出 |
| #11 禁止 Mock 实现 | → 红线四（禁止 Mock 实现）的执行引擎 |
