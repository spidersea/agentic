# 跨会话恢复流程

> 本流程按照 SKILL.md 第四章的上下文恢复协议，确保跨会话时不丢失关键状态。
> 触发方式: `/resume`

## 步骤

// turbo
1. **读取路由表**
   读取 `AGENT.md`，获取当前项目的路由规则和强制规则。

// turbo
2. **加载开发规范**
   读取 `.agent/skills/world_class_coding/SKILL.md`，恢复行为规范。

3. **定位检查点**
   - 搜索项目中最新的检查点文件或状态记录
   - 如果找到检查点：读取并解析检查点内容，确认：
     - 当前处于哪个 Phase
     - 已完成和待完成的工作
     - 正在修改的关键文件列表
   - 如果未找到检查点：**停下！** 向用户确认当前任务状态后再继续

3.5 **变更感知恢复（图谱增强）**
   - 📊 如果代码图谱可用：
     - 执行 `get_impact_radius(base="<检查点对应的commit>")` 
     - 获取自上次检查点以来的所有变更及其影响范围
     - 将变更摘要附加到恢复状态报告中：
       - 哪些文件被修改了
       - 修改影响了哪些模块和函数
       - 是否有与当前任务相关的变更
   - 如果图谱不可用：通过 `git diff` 查看检查点以来的文件变更
   - 这确保 Agent 不仅恢复了任务计划，还感知到代码库在检查点之后的真实变化

// turbo
4. **恢复任务上下文**
   - 读取验收契约 `{TaskName}_CONTRACT.md` 或实施计划
   - 读取检查点中列出的关键文件（≤5 个）
   - 对比检查点中的"待完成"清单，确认当前所处位置

5. **状态确认**
   - 向用户汇报恢复的状态：
     - "当前处于 Phase X"
     - "已完成: [清单]"
     - "待完成: [清单]"
     - "下一步: [具体行动]"
   - 等待用户确认后再继续工作

6. **继续执行**
   - 从确认的当前位置继续执行对应的 Phase
   - 严格遵循 SKILL.md 中该 Phase 的要求
