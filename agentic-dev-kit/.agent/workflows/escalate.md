# 手动压力升级

> 当 Agent 明显在原地打转或摸鱼但未触发自动检测时，用户可通过 `/escalate` 手动介入。
> 触发方式: `/escalate`

## 步骤

1. **加载 Escalation 技能**
   读取 `.agent/skills/escalation/SKILL.md`，理解 L1-L4 压力等级和七项检查清单。

2. **强制进入 L3 模式**
   无论当前失败计数，立即将压力等级设置为 **L3（强制清单）**：
   - 输出 `[L3 📋 — 用户手动触发]`
   - 必须完成**七项检查清单**（逐项确认）方可继续
   - 加载 `autoresearch/references/methodology-router.md`，评估是否需要切换方法论

3. **执行检查清单**
   逐项完成以下清单（每项必须有具体动作和输出）：
   - [ ] 逐字读完失败信号
   - [ ] 用工具搜索过核心问题
   - [ ] 读过失败位置的原始上下文（50 行）
   - [ ] 所有假设都用工具确认
   - [ ] 试过完全相反的假设
   - [ ] 能在最小范围内复现问题
   - [ ] 换过工具/方法/角度/技术栈

4. **汇报结果**
   清单完成后，向用户汇报：
   - 七项清单每项的执行结果
   - 当前方法论是否适合，是否建议切换
   - 如仍未解决：输出结构化失败报告（`escalation/SKILL.md` 格式）
