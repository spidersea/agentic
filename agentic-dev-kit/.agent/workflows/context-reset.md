# 上下文重置流程

> 当上下文被无关信息污染、会话过长、或需要切换任务时，使用本流程清理并恢复。
> 触发方式: `/context-reset`
>
> **融合理念**: 结合 CLAUDE.md 的 `/clear` 机制和本体系的上下文恢复协议。

## 应用场景

**被动场景（出现问题时）：**
- 🍳 **厨房水槽会话**: 一个会话中混杂了多个不相关任务，上下文充满无关信息
- 🔄 **反复纠正螺旋**: 纠正 Agent 两次仍犯错，上下文被失败方案污染
- 📦 **上下文溢出**: 会话已加载太多文件，AI 开始忽略指令或产出质量下降
- 🔀 **任务切换**: 完成一个任务后要切换到完全不相关的另一个任务

**主动场景（预防性重置，逻辑断点处主动执行）：**
> 来自 everything-claude-code 的 strategic-compact 理念：**在进入下一阶段前主动重置，而不是等待溢出**
- ✅ 调研/探索阶段完成后，**进入编码实现之前**（保留：spec 和 tasks.md；清理：调研过程的大量文件）
- ✅ 完成一个里程碑后，**开始下一个功能之前**
- ✅ Debug 完成后，**继续正常功能开发之前**
- ✅ 一个方向尝试失败后，**换新思路之前**

**⛔ 不应重置的时机**：
- 实现进行到一半时（会丢失变量名、文件路径、部分上下文状态）
- 正在处理依赖上下文推理的复杂 bug 时

## 自动压缩触发条件 (Auto-Compaction Rules)

> 借鉴 Claude Code 的 `auto_compaction_input_tokens_threshold = 200,000` 机制。
> 原则：**在逻辑断点处主动压缩，而不是等待溢出。**

**Agent 必须在以下条件满足时主动建议或执行上下文压缩：**

| 触发条件 | 动作 | 保留策略 |
|---|---|---|
| 对话轮次 > **15 轮** | 主动提醒用户考虑压缩 | 保留最近 8 轮完整上下文 |
| 累计加载文件总行数 > **5000 行** | 主动提醒并列出可卸载的文件 | 保留当前任务直接相关的 ≤5 个文件 |
| 单个 Phase 完成后 | 自动清理该 Phase 独有的文件缓存 | 保留产出物（spec/契约/检查点）+ 下一 Phase 需要的文件 |
| 连续 3 次工具调用结果被忽略（定义：Agent 连续收到 3 次工具输出后未在回复中引用任何一次结果） | 强制建议压缩（注意力衰减信号） | 保留最近 4 轮 + 任务契约 |
| 任务切换（从 A 任务到 B 任务） | 强制执行完整 `/context-reset` | 仅保留 AGENT.md + 新任务上下文 |

**压缩后恢复保证：**
- 压缩后必须保留：当前任务契约、最近检查点路径、活跃文件清单
- 压缩后必须重新执行「上下文恢复协议」步骤 4-6
- 压缩后必须向用户汇报：移除了哪些上下文、保留了哪些上下文

## 步骤
// turbo

1. **诊断上下文健康状态**
   评估当前上下文：
   - 当前会话是否混入了不相关的任务上下文？
   - 是否有反复失败的方案占用上下文空间？
   - 当前正在进行的任务是什么？是否需要保存进度？

2. **保存当前进度（如需要）**
   如果当前有未完成的任务：
   - 执行 `/checkpoint` 保存当前状态
   - 如果需要在新会话继续，执行 `/handoff` 生成交接备忘录
   - 如果任务不需要延续，确认用户同意放弃当前进度

3. **执行上下文清理**
   清除当前上下文中的无关信息：
   - 标记当前所有已加载的无关文件为"不再需要"
   - 清除失败方案的残余记忆
   - 重置回"干净"的工作状态

4. **按恢复协议重新加载**
   严格按照 SKILL.md 中的上下文恢复协议（第四章 4.3）执行：
   ```
   恢复序列:
   1. 读取 AGENT.md（获取路由和强制规则）
   2. 读取当前任务匹配的 .agent/rules/*.md（获取对应规则）
   3. 读取相关 SKILL.md（恢复行为规范）
   4. 读取最新检查点/状态文件（如有）
   5. 读取验收契约（如有）
   6. 加载当前任务直接相关的核心文件（≤5 个）
   ```
   - 📊 如果代码图谱可用，辅助步骤 6 的文件选择：
     - `get_impact_radius(changed_files=[当前任务涉及的文件])` — 精准选择最相关的 ≤5 个文件
     - 优先级：变更文件 > 直接调用方 > 测试文件

5. **确认恢复完成**
   向用户汇报：
   - 已清理的上下文内容摘要
   - 当前已加载的文件列表
   - 当前任务状态（继续之前的任务 / 准备新任务）
   - 提示用户可以开始新的指令


## 自动化合规验证
可以使用如下命令验证当前环境状态：
```bash
bash .agent/scripts/health-check.sh .
```

如果阻塞，可求助 `/debug` 流程。
