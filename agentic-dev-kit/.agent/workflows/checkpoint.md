# 生成检查点

> 按照 SKILL.md 第四章的检查点规范，记录当前任务状态。
> 触发方式: `/checkpoint`

## 步骤

1. **确认当前状态**
   - 确认当前所处的 Phase（1/2/3/4）
   - 确认当前任务名称
   - 确认关键文件列表

// turbo
2. **收集检查点数据**
   收集以下信息：
   - 当前时间戳
   - 当前 Phase 编号
   - 已完成的工作摘要
   - 待完成的工作清单
   - 当前正在修改的文件列表
   - 测试状态（通过/失败/未运行）
   - 本阶段做出的关键技术决策及理由
   - 明确的下一步行动

// turbo
3. **生成检查点文件**
   按以下模板生成检查点记录：

   ```markdown
   ## Checkpoint: CP-[编号] — [Phase 名称]
   - **时间**: [时间戳]
   - **当前 Phase**: Phase [1/2/3/4]
   - **已完成**: [已完成的工作摘要]
   - **待完成**: [剩余工作清单]
   - **关键文件**: [当前正在修改的文件列表]
   - **测试状态**: [通过/失败/未运行]
   - **决策记录**: [本阶段做出的关键技术决策及理由]
   - **下一步**: [明确的下一步行动]
   ```

   将检查点追加写入 task.md 或独立的状态文件中。

4. **确认记录**
   - 向用户确认检查点已保存
   - 提示用户可以安全地暂停或切换会话
