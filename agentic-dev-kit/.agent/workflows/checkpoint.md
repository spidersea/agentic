# 生成检查点

> 按照 SKILL.md 第四章的检查点规范，记录当前任务状态。
> 触发方式: `/checkpoint`

## 步骤

1. **确认当前状态**
   - **必须**确认当前所处的 Phase（1/2/3/4）
   - **必须**确认当前任务名称
   - **必须**确认关键文件列表
   - 如果上述任何一项无法确认，**禁止**跳过，必须回溯上下文或询问用户

// turbo
2. **收集检查点数据**
   **必须**收集以下全部信息（不可遗漏任何字段）：
   - 当前时间戳
   - 当前 Phase 编号
   - 已完成的工作摘要
   - 待完成的工作清单
   - 当前正在修改的文件列表
   - 测试状态（通过/失败/未运行）— **必须**基于实际命令输出，**禁止**凭记忆填写
   - 本阶段做出的关键技术决策及理由
   - 明确的下一步行动

// turbo
3. **生成检查点文件**
   **必须**按以下模板生成检查点记录，**禁止**省略任何字段：

   ```markdown
   ## Checkpoint: CP-[编号] — [Phase 名称]
   - **时间**: [时间戳]
   - **当前 Phase**: Phase [1/2/3/4]
   - **已完成**: [已完成的工作摘要]
   - **待完成**: [剩余工作清单]
   - **关键文件**: [当前正在修改的文件列表]
   - **测试状态**: [通过/失败/未运行]
   - **决策记录**: [本阶段做出的关键技术决策及理由]
   - **下一步**: [明确的下一步行动]
   - **autoresearch 产物**: [列出 security/debug/fix/ship/ 目录内最新的报告路径，无则填"无"]
   ```

   **必须**将检查点追加写入 task.md 或独立的状态文件中。**禁止**仅在对话中口头汇报而不写入磁盘。

4. **确认记录**
   - 向用户确认检查点已保存到磁盘
   - 提示用户可以安全地暂停或切换会话


## 自动化合规验证

bash .agent/scripts/health-check.sh .
