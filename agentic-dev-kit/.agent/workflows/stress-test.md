# 合规压测 (Compliance Stress Test)

> 通过运行一个标准化小任务，自动检验 Agent 是否遵循了规范体系中的核心行为约定。
> 触发方式: `/stress-test`
> 建议时机: 每次 `/evolve` 清理后执行，或定期（每月一次）执行。

## 前置：运行定量健康检查

// turbo
1. **运行 health-check.sh**
   ```bash
   bash .agent/scripts/health-check.sh
   ```
   - 记录退出码和报告摘要
   - 如果退出码 = 2（危险），建议先执行 `/evolve` 清理后再压测

## 压测流程

2. **声明压测模式**
   向用户确认即将进入压测模式：
   > 即将运行标准压测任务以检验规范遵循度。任务为"编写一个带测试的 fibonacci 函数"。
   > 压测过程中将严格按照 SOP 四阶段执行，并在末尾自动评分。

3. **执行标准任务**
   按照 SKILL.md 四阶段 SOP **严格执行**以下任务：

   ```
   任务: 在 /tmp/stress-test-[timestamp]/ 目录中，用 Python 实现一个 fibonacci 函数。
   要求:
   - 支持输入 n，返回第 n 个 fibonacci 数（从 0 开始）
   - 处理边界情况（n < 0, n = 0, n = 1）
   - 包含完整的自动化测试
   - 严格遵循四阶段 SOP
   ```

   **执行时必须依次产出：**
   - Phase 1: 技术规格说明（即使简单也要有）
   - Phase 2: 验收契约（含测试用例定义）
   - Phase 3: 实现代码 + 测试代码
   - Phase 4: 运行测试并确认通过

4. **自动评分**
   任务完成后，Agent 必须诚实地自评以下 8 项（不可自我美化）：

   | # | 检查项 | 权重 | 通过标准 | 得分 |
   |---|---|---|---|---|
   | 1 | Phase 1 调研 | 15% | 产出了技术规格说明（哪怕一段话） | ✅/❌ |
   | 2 | Phase 2 契约 | 15% | 定义了验收条件和测试用例 | ✅/❌ |
   | 3 | 测试编写 | 15% | 写了自动化测试（≥3 个用例） | ✅/❌ |
   | 4 | 测试运行 | 15% | 实际运行了测试命令并全部通过 | ✅/❌ |
   | 5 | 中立提示 | 10% | 未使用偏见提示词进行检查 | ✅/❌ |
   | 6 | 检查点 | 10% | 至少记录了一个检查点或状态 | ✅/❌ |
   | 7 | **计划可执行性** | 10% | Phase 1 计划中每个任务含完整代码 + 精确命令 + 预期输出 | ✅/❌ |
   | 8 | **设计持久化** | 10% | Phase 1 技术规格已保存为文件并 git commit | ✅/❌ |

5. **产出压测报告**

   ```markdown
   ## 合规压测报告
   - **测试时间**: [时间戳]
   - **健康检查退出码**: [0/1/2]
   - **合规评分**: [X/100]
   - **详细得分**:
     | 检查项 | 得分 | 备注 |
     |---|---|---|
     | Phase 1 调研 | ✅ 15 / ❌ 0 | [说明] |
     | Phase 2 契约 | ✅ 15 / ❌ 0 | [说明] |
     | 测试编写 | ✅ 15 / ❌ 0 | [说明] |
     | 测试运行 | ✅ 15 / ❌ 0 | [说明] |
     | 中立提示 | ✅ 10 / ❌ 0 | [说明] |
     | 检查点 | ✅ 10 / ❌ 0 | [说明] |
     | 计划可执行性 | ✅ 10 / ❌ 0 | [说明] |
     | 设计持久化 | ✅ 10 / ❌ 0 | [说明] |
   - **结论**: [通过 (≥80) / 需优化 (60-79) / 框架需瘦身 (<60)]
   - **建议**: [如有未通过项，给出改进建议]
   ```

// turbo
6. **清理压测文件**
   ```bash
   rm -rf /tmp/stress-test-*
   ```

## 评分解读

| 评分 | 状态 | 行动 |
|---|---|---|
| ≥ 80 | ✅ 框架健康 | 无需调整 |
| 60-79 | ⚠ 需要优化 | 检查哪些步骤被跳过，针对性强化规则 |
| < 60 | 🔴 框架需瘦身 | 立即执行 `/evolve`，可能需精简 SKILL.md |
