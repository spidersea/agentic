# 经验捕获工作流 (Skill Capture)

> **灵感来源**: [OpenSpace](https://github.com/HKUDS/OpenSpace) 的 Self-Evolving 机制
>
> 触发方式: `/skill:capture`
> 适用时机: 在你经过多次 Trial-and-Error (如大量调试、查阅文档或图谱分析) 成功解决了一个难题，或者完成了一个非常有参考价值的复杂功能闭环之后。

## 前置条件

- Agent 能够访问当前会话的最新的对话上下文。
- 目标目录 `.agent/skills/captured/` 已存在（如不存在请自行创建）。

## 步骤

0. **建立终点契约 (Task Contract)**
   > ⛔ **核心防御机制：对抗注意力衰减与产物断裂**
   - 收到 `/skill:capture` 的**第一秒钟**，你必须立即使用写文件工具，在工作区创建一个实体打卡文件（或整合进 `task.md` 中），强制列出本工作流的所有阶段作为 `[ ] TODO` 打卡项。
   - **终点拦截规则**：这是一个强状态契约。除非你亲手将该磁盘文件中的所有检查项全部标记为 `[x]`，否则**绝对不可**自行宣告任务完成。每完成下面的一步，你必须去物理更新该文件。

1. **回顾上下文 (Context Review)**
   - 自动扫描当前会话从该任务开始至今的交互记录。
   - 重点关注:
     - 遇到过哪些报错？
     - 进行了哪些关键的排障步骤测试？
     - 什么技术选型/逻辑设计被证明是行不通的（Dead Ends）？
     - 最终成功解决此问题的核心 Insight（突破点）是什么？

2. **提炼微型技能模式 (Pattern Extraction)**
   - 浓缩经验，总结出该类问题的一般化解决模式（Workflow/Checklist）。
   - 这个模式应该遵循 DRY 原则，剔除具体业务的专有数据，保留结构化框架。
   - 这不是长篇大论的教程，而是下一次 Agent 遇到类似情况时的“避坑防跑偏操作手册”。

3. **生成对应的 SKILL.md**
   - 给这个微型技能起一个具有辨识度、包含动宾结构且带有前缀的名字（例如 `captured-auth-token-refresh`、`captured-cypress-auth-bypass`）。
   - 在 `.agent/skills/captured/<skill-name>.md` 中生成如下内容的 Markdown：

   ```markdown
   ---
   name: <skill-name>
   description: 针对 [某种特定难题/场景] 的高频排雷指南与标准操作流
   # paths: ["可选，如果该技能仅在修改特定路径时生效"]
   ---

   # 捕获技能: <Skill Name>

   > 本技能从真实排障会话中捕获沉淀，旨在避免未来重复同样的试错成本。

   ## 1. 常见陷阱与错误路径 (Dead Ends)
   - 坑 1：[错误思路]，会导致 [报错/副作用]
   - 坑 2：[无效尝试]，已被证明在此架构下不通

   ## 2. 核心原理 (Root Cause & Insight)
   - [描述问题真正的底层原因，以及打破该困境的关键逻辑]

   ## 3. 标准操作规程 (SOP / Checklist)
   - [ ] 步骤 A：... (包含关键代码片段示例，非死板粘贴)
   - [ ] 步骤 B：... (附带必须同时检查的边界条件)
   - [ ] 验证 C：必须执行什么样的终端验证命令
   ```

// turbo
4. **生成路由说明**
   - 分析这个微型技能，提取 3-5 个**关键词**。
   - 在终端或总结中，向用户明确说明：请帮我在 `AGENT.md` 中增加路由规则，或者我直接帮您写入。例如：
     ```markdown
     | 遇到 [关键词 A、B] 相关问题时 | `.agent/skills/captured/<skill-name>.md` | 全文 |
     ```

5. **提交审核**
   - 将生成的 Markdown 路径和大致内容总结给用户。
   - 提示用户：这是我们在这个会话中新学会的技能，固化后即使新建会话，只要触发关键字即可复用此经验。
   - **检查终点契约**：确认已将物理打卡文件中的所有 [ ] 转化为 [x]，方可宣告 `/skill:capture` 流程完结。


## 自动化合规验证
可以使用如下命令验证当前环境状态：
```bash
bash .agent/scripts/health-check.sh .
```

如果阻塞，可求助 `/debug` 流程。
