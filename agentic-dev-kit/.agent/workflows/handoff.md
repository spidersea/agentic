# 生成交接备忘录

> 按照 SKILL.md 第四章的交接协议，生成精简的任务交接文件。
> 触发方式: `/handoff`

## 步骤

1. **收集当前状态**
   - 确认当前任务名称和所处 Phase
   - 确认最近的检查点状态（如有）
   - 整理已完成和待完成的工作
   - 先执行 `/checkpoint` 确保最新状态已保存

// turbo
2. **提炼核心背景**
   - 从当前会话中提取后继会话**最需要知道的核心上下文**
   - 严格控制在 5 句话以内
   - 只保留对继续工作至关重要的信息，删除一切冗余

// turbo
3. **整理关键文件**
   - 列出后继会话需要读取的文件（不超过 5 个）
   - 按优先级排序：契约 > 实施计划 > 核心代码 > 测试

// turbo
4. **记录已验证决策**
   - 列出本会话中已经做出并验证通过的技术决策
   - 这些决策后继会话不需要重新讨论

// turbo
5. **生成交接备忘录**
   按以下模板生成：

   ```markdown
   ## 任务交接备忘录
   - **任务名称**: [TaskName]
   - **当前状态**: [Phase X, CP-Y 已通过]
   - **交接时 git commit**: [git rev-parse --short HEAD 输出]
   - **核心背景** (≤5 句): [核心上下文摘要]
   - **关键文件**:
     1. [文件路径和说明]
     2. [文件路径和说明]
   - **已验证通过的决策**:
     - [决策1]
     - [决策2]
   - **待解决问题**:
     - [具体的、含完整命令的可执行下一步1]（如: `pytest tests/auth/ -v` → 预期 PASS）
     - [具体的、含完整命令的可执行下一步2]
   - **风险提示**: [如有未解决的阻塞项或注意事项]
   ```

// turbo
5.5 **执行会话结束钩子**
   ```bash
   bash .agent/scripts/session-end.sh
   ```
   自动保存 auto-checkpoint。如需提取本次会话的编码模式，可先运行 `/learn`。

6. **交接确认**
   - 将交接备忘录保存到项目中
   - 提示用户：在新会话中使用 `/resume` 即可安全恢复
