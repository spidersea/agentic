# Git 历史技能生成（Skill Creator）

> 灵感来源：[everything-claude-code](https://github.com/affaan-m/everything-claude-code) 的 `/skill-create` 命令。
> 核心理念：新项目冷启动时，通过分析 Git 提交历史自动提取项目特有的编码模式和惯例，生成 SKILL.md 草稿。

## 使用场景

- **新项目接入**：项目已有大量代码但没有编码规范文档，通过分析 Git 历史快速生成
- **规范更新**：定期从最近的 commits 中提取新模式，补充现有规范
- **团队对齐**：将团队高频使用的模式固化为 Skill

## 提取流程

### 1. 分析范围配置

```bash
# 默认：最近 100 个 commits
git log -100 --format='%H %s' --no-merges

# 或指定时间范围
git log --since="3 months ago" --format='%H %s' --no-merges
```

### 2. 模式识别维度

| 维度 | 提取方法 | 产出 |
|---|---|---|
| **命名惯例** | 分析文件名、函数名、变量名的模式 | 命名规范规则 |
| **目录结构** | 分析目录组织模式 | 项目结构约定 |
| **测试模式** | 分析测试文件的结构和覆盖率模式 | 测试编写规范 |
| **提交风格** | 分析 commit message 格式 | Git 工作流规则 |
| **代码模式** | 分析高频出现的代码结构（错误处理、日志…） | 编码规范 |
| **依赖使用** | 分析常用库的使用模式 | 依赖使用指南 |

### 3. 产出

生成 SKILL.md 草稿，包含：
- 项目概述（技术栈、目录结构）
- 提取的编码规范（带代码示例）
- 测试约定
- Git 工作流约定

### 4. 人工审核

⚠️ 生成的 Skill 是**草稿**，必须人工审核后才能纳入正式技能库：
- 确认提取的模式确实是想要保留的
- 删除临时性或过时的模式
- 补充提取器无法发现的隐式约定

## 与本能系统的协作

`/skill-create` 也可同时生成本能（instinct）数据：
- 从 commit 模式中提取的规律 → 以 confidence=2 写入 `.agent/instincts/pending.yml`
- 需要在后续使用中被验证才能升级

## 限制

- 只能提取**代码层面**的模式，无法提取设计决策背后的原因
- 对小型项目（< 50 commits）效果有限
- 无法识别反模式（哪些做法是错误的）——这需要人工标注
