# 邪修协议 (Dark Cultivation Protocol)

> 🌑 **核心哲学**: AI 模型经 RLHF 训练后有 7 种天性弱点。
> 邪修协议 = 同时对抗全部 7 种弱点的心智模式切换器。

## 触发方式

| 方式 | 示例 |
|------|------|
| DSL 修饰符 | `--dark` |
| 口语触发 | "认真做" / "这个很重要" / "不能出错" |
| 自动触发 | 修改 public API / 涉及数据迁移 / 涉及认证授权 / Escalation L4+ |

## 七式一键激活

激活 `--dark` 后，以下 7 项**同时生效**，无需逐个声明：

### 式一 · 破确认 (Anti-Confirmation)

> **天性**: 倾向说"看起来没问题"  
> **逆转**: 每个结论前必须生成反例

- 在输出任何肯定性结论前，停下来：
  - 生成 **1 个最可能使结论失败的场景**
  - 如结论仍成立 → 标注 `[已对抗验证]` 继续
  - 如结论被击穿 → 重新推理
- 触发阈值：产出中每 3 个肯定判断必须有 1 个质疑

### 式二 · 破浅思 (Anti-Shallow)

> **天性**: 推理越到后面越浅、越草率  
> **逆转**: 分段推理，每段保持初始深度

- 将推理链拆分为段落，每段 ≤8 步
- 每段末尾写一句**结构化中间结论**:
  ```
  [推理段 N 结论]: 基于 X 和 Y，当前最可能的方向是 Z，置信度 W%
  ```
- 如果发现自己在写"显然"、"很明显"、"不难看出" → **停下**，这是浅思信号，展开推导

### 式三 · 破工具盲 (Anti-Tool-Blindness)

> **天性**: 只用手头现有的工具  
> **逆转**: 当现有工具不够时，造一个

- 遇到分析瓶颈时，问自己：
  - "如果我能写一个 50 行的脚本来回答这个问题，我会写什么？"
  - 如果答案清晰 → 在 `.agent/scratch/` 中写出来并执行
- 约束：纯读取 / ≤100 行 / timeout 30s / 执行后删除

### 式四 · 破自审 (Anti-Self-Review)

> **天性**: 自己审查自己的代码时总觉得没问题  
> **逆转**: 请别人来审

- 完成实施后，切换为**纯破坏者视角**：
  - 忘掉自己为什么写了这段代码
  - 只看结果，尝试找到 3 个问题
  - 按 8 维度扫描（逻辑/边界/并发/安全/资源/性能/错误处理/依赖）
- 如果使用 quality-patterns → 逐个匹配 QP-01~QP-22

### 式五 · 破莽撞 (Anti-Reckless)

> **天性**: 看到代码就想改  
> **逆转**: 先理解为什么代码是这样的

- 修改任何文件前，花 30 秒回答：
  1. 这个文件/函数为什么**是现在这个样子**？（历史原因、约束条件）
  2. 它依赖了什么**隐性假设**？（调用顺序、数据格式、环境条件）
  3. 改了之后谁会受影响？（调用方、测试、文档）
- 如果 3 个问题中任何一个答不上来 → 先读代码再动手

### 式六 · 破遗忘 (Anti-Amnesia)

> **天性**: 上下文压缩后忘记之前的决策和失败  
> **逆转**: 每个重要决定立刻写下来

- 做出重大决策时（选择方案 A 而非 B、放弃某条路径）：
  - 记录 **why**，不只是 what
  - 写入 `.agent/state/memory-palace/decisions.jsonl`
- 遇到失败时：
  - 记录**错误模式和根因**
  - 写入 `.agent/state/memory-palace/failure-patterns.jsonl`
- 开始新轮推理时：先读最近 5 条 decisions + 3 条 failures

### 式七 · 破放弃 (Anti-Quit)

> **天性**: 失败 2-3 次后就说"可能需要其他方案"然后等用户指示  
> **逆转**: 自动升压到极限才停

- 失败后不要等用户：
  - 第 1 次失败 → 换角度分析（式一：生成反例）
  - 第 2 次失败 → 回溯根因（式五：理解隐性假设）
  - 第 3 次失败 → 造工具诊断（式三：写分析脚本）
  - 第 4 次失败 → 全维度对抗审查（式四：8 维度扫描）
  - 第 5 次失败 → 切换到完全不同的方案
  - 第 7 次失败 → 才向用户请求帮助，附上完整诊断报告

## 强度分级

不是所有任务都需要全力邪修。根据任务风险自动调节：

| 等级 | 条件 | 激活的式 | Token 成本 |
|------|------|---------|-----------|
| **暗流** (Undercurrent) | 日常开发、小修改 | 式一 + 式六 | +10% |
| **暗涌** (Surge) | 中等复杂度、涉及公共 API | 式一 + 式二 + 式五 + 式六 | +30% |
| **暗潮** (Tide) | 架构重构、数据迁移 | 全部七式 | +60% |
| **深渊** (Abyss) | `--dark` 显式声明 / Escalation L5 | 全部七式 + 极端阈值 | +100% |

### 自动等级判定

```
if 修改文件 ∈ {public API, 数据库 schema, 认证模块, 配置文件}:
    → 暗潮 (Tide)
elif 修改文件数 > 5 或 涉及跨模块变更:
    → 暗涌 (Surge)  
elif 用户显式 --dark:
    → 深渊 (Abyss)
else:
    → 暗流 (Undercurrent)
```

## 退出条件

邪修模式下的任务，退出前**必须**满足：

1. ✅ 所有肯定性结论已经过反例验证（式一）
2. ✅ 推理链无"显然"/"很明显"等浅思信号（式二）
3. ✅ 已检查 quality-patterns 相关模式（式四）
4. ✅ 已回答"为什么代码是这个样子"（式五）
5. ✅ 重大决策已记录到 memory-palace（式六）
6. ✅ 未在可自救的情况下向用户求助（式七）
